## Crash fixed

**`/BuyNow/Forward`** — `src/modules/SellingPlatform/BuyNow/Forwards/page.jsx:54`

```
forwardInstruments[0].MarketPlatformInstrumentId
// crash: Cannot read properties of undefined (reading 'MarketPlatformInstrumentId')
```

Root cause is upstream of `orderEntry.ts`: `forwardInstruments` is derived in `ForwardsContext` (line 116-119) by filtering `marketPlatformInstruments?.Data` on `TradeTypeMeaning === 'Forward'`. The previous fixture returned `Data: []`, so `forwardInstruments` was an empty array, and the page's `useEffect` indexed `[0]` unconditionally.

Fix: populated `marketPlatformInstrumentsFixture.Data` with one Prompt (id 101) and two Forward (ids 201, 202) instruments. `marketPlatformInstruments.ts` was not claimed by any team in the manifest and is the only place the field can be supplied — `orderEntry.ts` cannot fix this crash because `GetOrderEntryData` is a downstream call triggered after a grid cell selection. Flagging here in case the orchestrator wants to reassign ownership.
