## Infra change rationale
Bootstrap previously called factories with no arguments (`factory()`), making it impossible to distinguish e.g. `EntityReport/Read` calls for `TradeEntryValuation` vs `TradeEntryValuationMissingPrices` vs `IntegrationStatus` — all three share the same path. Now the registry passes `(path, init)` through; factories that don't care still ignore both. The EntityReport dispatcher parses `init.body` (JSON-stringified by excalibrr's `api.post`) and switches on `body.ReportName`. Adding a new EntityReport-driven page is now: 1) add a Schema entry to `SCHEMAS`, 2) add a row dataset to `READ_DATA`. No new endpoint key needed.

This also resolved a latent bug where `index.ts` had two `'EntityReport/Read'` entries (T3 and the older T7 stub), the second silently shadowing the first; T3's `IntegrationStatus` would have been broken once Valuations went live. Now both flow through the central dispatcher.

## Consumer crashes fixed
- `src/api/useContractsReport/index.ts:23` — `resp.Data.map(...)` on `Details[]` requires non-empty `Details` per row. Each contract in the new `contractsReportFixture` has 1–2 details with `FromLocationName`, `ProductName`, `Quantity`. Without this, `getSortedProductGroups` (line 46) would produce an empty product summary and `Locations: [...new Set(...)]` would be empty.
- `src/hooks/useEntityReport.ts:33,53` — schema and read both call `api.post('EntityReport/{GetSchema,Read}', { ReportName, ... })`. Previously the page key (e.g., `TradeEntryValuation`) was ignored and OnlineOrders' schema came back for every report, breaking `getServerParams` and the column factory. Now each report gets its own schema with relevant `EntityViewColumns`.
- `src/modules/ContractManagement/ContractRevaluation/page.tsx:43` — `metadata?.Data` reads `CounterParties/Locations/Products/ValidationRules`. Empty fixture would have rendered the page but with a non-functional date picker (no validation rules) and no dropdown options. Now fully populated.
- `src/modules/FormulaTemplates/FormulaTemplatesCore.tsx:37` — `formulaTemplates?.Data` is required for the grid; metadata drives the create/edit drawer dropdowns. Both populated.
