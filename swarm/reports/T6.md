## CommandCenter PageView Layout pattern (reference for other teams)

The actual `Metadata` consumer (`GridSettingsDrawer.tsx`) reads `metadata?.Data.LocationTree`, `ProductTree`, `QuoteConfigurations` — those fields are required and populated.

The brief's `Columns / Filters / Sortable / PageView.Layout` scaffold is included alongside as forward-compatible structure, even though `CommandCenterPage.tsx` currently hard-codes its 4 widgets in `widgets` (lines 178–233) rather than driving them from metadata. Shape used:

```ts
PageView: {
  ViewName: 'Default',
  Layout: [
    {
      WidgetId: 10,                                // 10–20 reserved per brief
      WidgetType: 'IntradayCompetitorMovement',    // matches WidgetGridSettings keys in pageViewTypes.schema.ts
      Title: 'Intraday Competitor Movement',       // matches WidgetTitle union in types.schema.ts
      StorageKey: 'CommandCenter-IntradayCompetitorMovement',
      Endpoint: 'CommandCenter/GetIntradayCompetitorMovementData',
      Position: { Row, Col, RowSpan, ColSpan },
    },
    // …StrategyMiss (11), MarginSummary (12), VolumePace (13)
  ],
}
```

Saved page views (`AllPageViewResponseData[]`) follow `UpsertPageViewPayload` from `pageViewTypes.schema.ts`:
- `userPreferenceId: number`
- `display: string`
- `pageServerSideFilters: { LocationHierarchyTypeCvId, ProductHierarchyTypeCvId }`
- `widgetSettings: WidgetSetting[]` with `name` (storage key like `CommandCenter-MarginSummary`), `gridConfigSettings: { filters: [], columns: [] }`, and `widgetGridSettings: { [WidgetTypeKey]: { thresholds, filters } }` — note the bracket-keyed shape is required because `util.ts:106` does `widgetSetting.widgetGridSettings[key]` where `key = name.replace('CommandCenter-', '')`.
