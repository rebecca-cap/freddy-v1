## Consumer notes / shape findings

- **AvailabilityMaintenance**: response is `GridResponse = { Rows, MaxPromptUpdateFromAllocationDateTime }` — NOT wrapped in a `Data` envelope. Consumer at `src/modules/Admin/AvailabilityMaintenance/components/VolumeManagement/index.tsx:132` calls `data?.Rows`. Fixture matches.
- **QuantityDistribution**: `APIResponse<{ Rows, PeriodDisplayMappings }>` — Type B-ish (`Data.Rows`), but with `PeriodDisplayMappings` as a peer of `Rows` inside `Data`. Confirmed in `src/modules/Admin/ManageQuantityDistribution/api/type.schema.ts:31`.
- **OrderDashboard `/` route**: `useOrderDashboard` calls the same `GetPendingOrders` endpoint twice (Prompt + Forward) and the same `GetRecentlyProcessedOrders` twice. Our registry returns the same fixture for both calls; the consumer concatenates `pendingPromptOrders.Data + pendingForwardOrders.Data` so visually both grids will show identical rows. Acceptable for a render-doesn-crash bar; flag for verify swarm if "Pending Forward" needs distinct rows.
- **FormulaTemplates split**: per `src/modules/FormulaTemplates/Api/FormulaTemplateEndpoints.ts`, T7's PE variant uses `FormulaTemplateManagement/*` keys; T10 owns the OSP variant `MarketPlatform/ContractTemplates/*`. No collision.
- **CalculatedValueReport split**: existing `calculatedValueReport.ts` (T7) owns `ReadCalculatedValues`. T10 added only `ReadMetadata` and `GetSecondaryPriceBreakdown`, in a separate file `adminCalculatedValueReport.ts`.
