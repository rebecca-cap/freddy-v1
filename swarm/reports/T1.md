## Consumer crashes fixed

`/Admin/Users` — `src/modules/Admin/ManageUsers/components/colDefs.tsx:295`
> `Cannot read properties of undefined (reading 'find')`

Root cause: `userMetadata?.Data?.IdentityProvidersList.find(...)` — the optional chain stops at `Data`, but `.find` is unconditional on `IdentityProvidersList`. Old fixture had no `IdentityProvidersList` key. Added it (3 providers: Local, Azure AD, Okta) plus the related lists also unconditionally read elsewhere in the file:
- `ContactMethodsList` (colDefs.tsx:313)
- `QuoteConfigurationMappingGroups` (colDefs.tsx:340, 348)
- `CounterPartyList` (colDefs.tsx:89, used optional-chained but read in bulk editor)
- `ImpersonationCounterPartyList` (used by `getImpersonationtionCounterPartyList` util)

Each user row also now carries every field referenced by colDefs (`IdentityProviderId`, `PreferredContactMethodCvId`, `QuoteConfigurationMappingGroups`, `Roles`, `CounterPartyAssociations`, `IsApproved`, `IsLocked`, `IsDisabled`, `LastLoginDateTime`, `LoginCount`, `IsOptedOutOfMarketingNotifications`, `ExternalId`, `CounterPartyId`).
