## Consumer crashes fixed

`src/modules/Admin/ManageOpisCurves/components/Grid/ManageOpisCurvesGrid.tsx:57` —
```
opisMetadata?.ExchangeSymbolList.filter((x) => !selectedExchangeSymbols.has(x))
```
The previous `opisMetadataFixture` returned `{ Data: { Locations, Products, Frequencies } }`, but `OpisMetadataResponse` is read top-level (no `Data` wrapper) and requires `ExchangeSymbolList: string[]`. The new fixture returns top-level `ExchangeSymbolList` plus the metadata extras, so the optional chain succeeds and `.filter` runs against an array. Manifest reported the line as 40; the actual crash site in the current consumer is line 57 — same root cause.

## Envelope decisions (verified against consumers)

- **Type A** `{ Data: T[] }`: pricePublishers (`Data.map`), opisCurves (`Data?.map`), priceConfigurationRead, priceNotifications products/locations/recipientData/subscriptions, priceTranslations (`Data.map` / `Data.filter` in mutations).
- **Top-level (no envelope)**: `opisMetadataFixture` exposes `ExchangeSymbolList` directly; `priceTranslationsMetadataFixture` exposes `LocationSourceValues` etc. directly; `priceImportMappingsMetadataFixture` exposes `Products`/`Locations`/`CounterParties`/`PricePublishers` directly. These match the typed `useQuery<...>` signatures in `useDTNMappings`, `usePriceTranslations`, `usePriceConfigurations`.
- **Nested `{ Data: { ... } }`**: priceImportMappings (`{Data: { LocationRules, ProductRules, SupplierRules }}`), priceConfiguration metadata (`{Data: { LocationGroupList, ProductGroupList }}` per `ProductGroups/index.tsx`), priceNotifications metadata (`APIResponse<MetadataData>`).
