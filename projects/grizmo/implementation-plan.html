Why

What problem this solves

When Frank opens /p/grizmo, none of his shipped Grizmo work shows up — the page looks like an empty project. This wires the inert features so the Hub surfaces real progress.

The gap

The Hi-Fi and Lo-Fi timelines were grey "coming soon" boxes. External Resources had six disabled chips. Frank's actual Grizmo lo-fi wireframes live outside the project tree at ~/Documents/Claude/Projects/Ai Opportunities/, with no way for the Hub to surface them.

The four cross-cutting decisions
  • Filesystem-first over a database — JSON files Frank can ls, git diff, and hand-edit.
  • Wireframe DS via sandboxed iframe only — zero style leakage into the host app.
  • Pointer files (externals.json) over copies (go stale) or symlinks (break Finder, confuse provenance).
  • Per-commit Launch is deferred to Phase 3 — Phase 1 Launch just opens the live dev URL.

Step 1b

Lo-Fi external rounds via externals.json

Done

Surface lo-fi wireframes that live outside the project tree through a pointer file, rendered in a sandboxed iframe — no copies, no symlinks.

Why

The Grizmo lo-fis sit at ~/Documents/Claude/Projects/Ai Opportunities/. Copies go stale silently; symlinks break Finder previews and blur "what's mine" inside the project. A pointer file (externals.json) is hand-editable, diffable, and points at the source of truth.

Outcome (definition of done)
  • listLofiRounds() appends external entries (marked external: true) after in-tree rounds.
  • Each external row shows an "External" pill, the label, and a file:// open link.
  • A collapsed iframe preview toggle (sandbox="allow-same-origin", ~600px tall) renders the wireframe in native styling.
  • Path-traversal outside the pointed-at directory is rejected by the serving route.
Prompt → Claude Code · Step 1b

The pointer file lives at ~/freddy/projects/{slug}/design-system/lofi/externals.json and is a JSON array of { label, path, indexHtml } entries (path is an absolute directory outside the project; indexHtml is the entry file inside it).
