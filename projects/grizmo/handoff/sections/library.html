Library, Save & Share

The Library is where reusable questions live, and where in-the-moment conversations get promoted into durable, shareable assets. It holds two kinds of saved item and gives the analyst a calm path from "I just asked this twice" to "this is now a one-click routine my team can run."

Two kinds of saved item

Every saved item is either a Prompt — a single reusable question — or a Path — an ordered list of questions that run in sequence. Both carry a name, a description, a scope (row-level versus aggregate/whole-view), a subject tag, a star for favoriting, and optional sharing.

Library cards, search & filters

The Library is list-first. A toolbar across the top offers a search field, a starred-only toggle, and an ownership filter with three states: All Yours Shared with you (see 1). Each card stacks a title with an actions row, then a meta row of pills — a kind pill reading Path · N steps or Prompt alongside scope and subject (see 2) — then a description clamped to two lines (see 5), then a footer carrying provenance on the left and the primary action on the right. Edit and Share reveal on hover; the star stays visible at all times (see 4).

{{figure:c2-library}}

Two empty states cover the surface: a "nothing saved in this tab" state when a tab has no items, and a "no matches" state when filters or search exclude everything, paired with a Clear filters affordance.

Running: Run vs Open

The card footer's action depends on the item kind (see 3). Run executes a path's steps sequentially — Scout switches to the chat view and streams each answer in turn. Open drops a prompt's text into a new ask, ready to send. A row-scoped item carries a guard: if it is run with no row in context, Scout surfaces a warning to Pick a row first rather than answering against the wrong scope.

The Save modal

Saving opens a two-tab modal. The Details tab collects a required name, an optional description, a subject, and the step list. Save stays disabled until there is both a name and at least one kept step.

{{figure:c2-save-details}}

The Share tab carries an audience picker built from scope cards — Only me Team Specific people — with a scrollable avatar list revealed when "Specific people" is chosen. The same picker control powers both this tab and the standalone Share modal, so the audience experience is identical in both places.

{{figure:c2-save-share}}

The inline path editor

Within the Details tab, a path's steps are directly editable: each step can be renamed inline, reordered up or down, or dropped. A dropped step renders struck-through and can be restored rather than vanishing. A Flatten on save option collapses the whole sequence into a single prompt for analysts who want one question instead of a multi-step run.

{{figure:c2-path-editor}}

The path-candidate nudge

After the analyst runs two or more questions of the same scope in a row, Scout offers an inline prompt beneath the latest answer: Save these N questions on this row as a path? (or "on the quote book" for whole-view scope). It previews the step labels, offers a Save as path button, and a dismiss control.

{{figure:c2-path-candidate}}
Appears once per qualifying run. The nudge collapses after saving or dismissing, and questions already saved are excluded from later candidates so the same group is never re-suggested.

The standalone Share modal

Opened from a card's overflow menu, the Share modal headers with the item name (see 1), presents the shared audience picker (see 2), and confirms with a Share action (see 3). One difference from the Save modal's Share tab: the standalone modal omits the "Only me" option, since sharing nothing privately is not a meaningful action here.
