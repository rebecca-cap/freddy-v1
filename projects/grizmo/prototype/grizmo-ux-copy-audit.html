01Cross-cutting themes

Most individual findings ladder up to five recurring patterns. Fixing these at the theme level resolves the long tail below and is where the biggest clarity gains live.

Terminology fragmentation highest impact

The same concept is named differently across surfaces, so users can't tell that the legend hint, the context chips, and the modal badge all describe one thing.

  • Scoping a question: scope Grizmo · set context · Row-scoped · this row
  • Reusable sequence: path · multi-step investigation · walkthrough
  • The conversation: session · conversation · chat · your context
  • Closing: Close · End session · discard · Keep going
  • Saving an item: Bookmark vs Saved (same button)

Unit convention violated house standard

Gravitate's standard is decimal dollars ($0.0100/gal), never cents. Grizmo's answer bodies, follow-ups, saved prompts, and activity feed use ¢ throughout — and sometimes mix ¢, $/gal, and bps in one sentence.

  • Pervasive in fakeLlm.ts answer bodies & action chips
  • Baked into reusable saved prompts users re-read
  • Bare prices with no unit (2.6620) clash with $1.9420/gal elsewhere

Onboarding & discovery gaps P0 cluster

The moments meant to teach the product don't. New users can't learn what Grizmo does, what a "path" is, or how to populate the library.

  • Intro bubble: "Your new pricing co-pilot. Click the G" — never says what it answers, and "the G" doesn't match the visible Ask Grizmo button
  • "path" is the central feature but is never defined before first use
  • Paths empty state tells users to bookmark to populate it — but paths are created via Save as path

Destructive actions under-explained trust/safety

Consequential controls name the object but not the consequence or recoverability, and the close flow uses four verbs for two outcomes.

  • Close confirm: title "End this session?" vs button "Close & discard" — mismatched
  • Clear conversation doesn't mention Undo exists
  • Apply-price action chips don't state scope (which/how many rows)

Tooltip / aria-label drift & placeholder-only labels a11y + polish

Header controls' tooltips drop the object ("Minimize", "End session") while their aria-labels add it ("Minimize Grizmo", "End Grizmo session") — an inconsistent pattern across the five controls. Several inputs (feedback note, path name/description) rely on placeholders as their only label, which disappear on focus and aren't announced to screen readers. Confirmation toasts vary in register for one flow: Thanks · Saved · Feedback received · Feedback logged.
