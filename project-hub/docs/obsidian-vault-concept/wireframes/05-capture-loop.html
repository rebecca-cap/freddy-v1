Move 4 · Stay populated

The vault fills itself

A second brain only works if it stays fed. This move makes capture a side effect of work you already do — promoting a project, running a PD sync, letting Claude Code build something. Notes get created and updated automatically, so the vault doesn't depend on anyone remembering to write things down. Then the captured knowledge loops back into the Hub and informs the next project.

4 · Back in the Hub

📚 Obsidian KB page 03

The whole vault, browsable read-only inside the Hub — fresh on every load.

🔗 Project → Knowledge 04

Each project shows its linked Decisions / Patterns / Learnings — which inform the next project.

Single write-path rule (research). Write to files on disk when Obsidian may be closed; use the Local REST API only when it's guaranteed open. Never both at once, or Obsidian's in-memory copy clobbers the disk edits.
Why this matters. Obsidian's #1 failure mode is silent rot — notes nobody maintains. Making capture a side effect of work you already do is the antidote: the vault stays current without anyone treating it as a chore.
