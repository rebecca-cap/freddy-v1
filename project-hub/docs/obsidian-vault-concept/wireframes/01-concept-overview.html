Project Hub × Obsidian Vault

Two surfaces, one brain

The Hub and your Obsidian vault hold the same product-design knowledge in two modes. The Hub is the operational surface — projects, prototypes, the lo-fi→hi-fi→handoff flow, generated design-system specs. The vault is the cognitive surface — decisions, patterns, learnings, and the linked graph that connects them. Today the vault is a private silo on one machine. Move it into the shared repo and bridge the two, and the team inherits the why behind every project — not just the what.

Who owns what — split by volatility, not by topic

The one architectural trap (from the research): plain markdown rots as a design-system spec because it can't sync token/component values from code — "Stale Documentation Syndrome." So the vault owns the durable, narrative knowledge and links to the Hub's already-generated specs as the single source of truth. Each does the job it's actually good at.

🧠 Vault owns — durable & narrative

Decisions — ADR-style: why we chose X, what we rejected
Patterns — how-to / reusable conventions
Learnings — gotchas, root-cause, how-to-avoid
Projects · Retros · Syncs — the story & the graph
Design-system narrative — principles, do/don't, intent

🗂️ Hub owns — generated & live

Token / component specs — values from the JSON source
Interactive specimens — /design-system, /wireframe-system
Runtime — which prototypes are running, on which port
Project lifecycle — phase, versions, covers, handoff
Vault DS notes link to these instead of restating values → no drift.
