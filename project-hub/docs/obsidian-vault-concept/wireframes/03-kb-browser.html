Tailwind Merge Class Conflicts

Problem

After a build, tailwind-merge silently stripped color and font-size classes off some components — text rendered at the wrong size and lost its color with no error.

Root Cause

Arbitrary values like text-[var(--gds-font-size-lg)] were read as part of the textColor group, so merge treated font-size and color as conflicting and dropped one.

Fix

Replaced arbitrary text-[…] utilities with named classes that map to the tokens. No more group collision; merge keeps both.

How it works — an Express route fast-globs /vault, parses each note with gray-matter (YAML frontmatter + body), and serves JSON. markdown-it plus a small wikilink plugin render [[links]] as in-Hub /kb/:slug anchors. No Obsidian install or running app required to read.
Why this matters — graceful degradation — engineers and PMs can read the second brain right in the browser. The Open in Obsidian deep-link (obsidian://) is there for anyone who wants the full graph + editing — but it's optional, not a prerequisite.
