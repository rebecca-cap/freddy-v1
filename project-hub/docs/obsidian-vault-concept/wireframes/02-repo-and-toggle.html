Move 1 · Co-locate

The vault lives in the repo

Drop the vault into the same GitHub repo that already ships the Hub app. One git clone delivers both — the app and the knowledge. Each teammate opens the vault/ folder as a second vault in their Obsidian and toggles between their own notes and the shared Project Hub vault with a hotkey. No new tool, no separate sync service: the team's normal commit/push flow becomes the sync path.

What gets committed — an allowlist, not the whole config

.gitignore — the allowlist

Ignore .obsidian/ wholesale, then re-include just the team-shared files.

# vault/.gitignore
# ignore the whole config dir…
.obsidian/*

# …then re-include the shared set
!.obsidian/app.json
!.obsidian/appearance.json
!.obsidian/community-plugins.json
!.obsidian/core-plugins.json
!.obsidian/hotkeys.json
!.obsidian/snippets/

# always keep these OUT (per-person / churny)
.obsidian/workspace.json
.obsidian/workspace-mobile.json
.obsidian/cache
.trash/
.DS_Store
Why an allowlist? workspace.json is per-person pane & layout state — which notes you have open, where your splits are. It changes constantly and would conflict on every single sync. So it stays ignored.

The re-included files ship one consistent theme + plugin set to the whole team: open the vault and you get the same appearance, the same enabled plugins, the same hotkeys — no per-person setup. Commit the shared brain, ignore the personal desk.

Onboarding — three steps, in the repo README

Onboarding (in the repo README)

Lives at vault/README.md — first thing a teammate sees.

1
Clone the repo — you already do this for the Hub app. The vault came along for free in /vault.
2
In Obsidian → Open folder as vault → pick ~/freddy/project-hub/vault. Theme + plugins + hotkeys come pre-set from the allowlist.
3
Toggle anytime between your own vault and the Project Hub vault. Read, link, edit — then commit & push like any other repo change.
Important — don't run the Obsidian Git plugin here. The research turned up a hard constraint: the Git plugin requires vault-root == git-root ("one repo = one vault"). In this layout the vault is a subfolder (project-hub/vault), not the repo root — so the Git plugin won't work for it and shouldn't be installed. That's fine: the repo's normal commit/push — the exact flow the team already uses for Hub code — is the sync path. No extra plumbing.

No license blocker either: Obsidian has been free for work use since Feb 2025, so every teammate can open the shared vault at no cost.
