Moves 5 & 6 · Compounding

The graph is the institutional memory

The linked graph turns scattered notes into a navigable map of how the product got here — which decisions and patterns each project leans on, and which ones quietly get reused everywhere. Conforming the Decisions folder to ADRs (one decision per file, numbered, immutable once accepted) makes that map trustworthy: you can hand it to a new teammate and they'll believe it.

Why this matters. The shared node is the payoff. A new teammate opens the graph, sees that one decision or pattern touches half the portfolio, and knows where to start reading. Reuse that lived only in someone's head becomes a node anyone can click.

The rules that make the log trustworthy:

1 One decision per file. No mega-docs — each ADR answers exactly one question.
2 Numbered NNNN-title.md — order is the timeline, no reshuffling.
3 Immutable once accepted. Don't edit history — write a new ADR that supersedes: the old one.
4 Reviewed in a PR, like code. A decision lands the same way a change does.

That audit trail is what turns the log into trustworthy memory — and kills the bus-factor. The why survives the person who decided it.

Hygiene — pick a small, controlled tag vocabulary and prune it in retros. Shared vaults rot faster than personal ones because no single owner feels responsible; the ADR rules and a tight tag set are the guardrails that keep the graph readable a year from now.
