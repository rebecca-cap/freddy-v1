## Why this exists

The Hub's **Prototypes tab** already has a real, code-centric **hi-fi** system: variants are git worktrees on branches, "launch" boots a dev server on a port, and "promote" is a `git merge` + version stamp. But **lo-fi is only a placeholder** — `TimelinePlaceholder.tsx` shows disabled mock chips, and `LofiRounds.tsx` (in Resources) can only make flat `round-N/` folders and open their `index.html`. There is no lineage between rounds, no branch-options, no per-round context or decisions log, no link to a shared design system, and no lo-fi → hi-fi handoff.

The goal is to make the lo-fi section a first-class, **document-centric** workspace that mirrors how design actually happens: a kickoff, then iterative rounds with a PM, branching parallel options when directions diverge, logging decisions transparently — and finally handing off to hi-fi **with all that context auto-wired**, so Claude never has to be manually pointed at the lo-fi work again.

## Entities

| Entity | What it is | On disk |
|---|---|---|
| **Project (lo-fi side)** | A design exploration. | `~/freddy/projects/<slug>/`, lo-fi workspace at `design-system/lofi/` |
| **Kickoff** | The brief / kickoff-meeting notes that seed round 1. | `lofi/KICKOFF.md` |
| **Round** | A *sequential* iteration of the UX (R1→R2→R3). One folder each, referencing its predecessor. Holds the canonical wireframe(s) for that pass. ≈ "one pass with the PM." | `lofi/round-N/` |
| **Branch-option** | A *parallel* alternative explored off a round (Direction A/B/C). Compared, then the chosen one folds forward into the next round. **Not a git branch — sibling folders.** | `lofi/round-N/options/<opt>/` |
| **Round guide** | The transparency artifact — generated by `/wireframe-index`. Catalogs screens + *Changes from Previous Round* + *Design Decisions* + *Open Questions*. This is the changelog/decisions layer. | `round-N/WIREFRAME-GUIDE.html` |
| **Context folder** | Where Frank drops transcripts/scripts; the skill mines these for the "why" when writing the guide. | `round-N/context/` |
| **Round CLAUDE.md** | The per-round working agreement = "the link to give to Claude." Points at the design system, the `wireframe-index` skill, the previous round, the kickoff, and `resources/`. | `round-N/CLAUDE.md` |

### Rounds vs branch-options (the key distinction)
- **Rounds are sequential.** R2 builds on R1; R3 builds on R2. Each round's `CLAUDE.md` points back at the previous round as the base to iterate on.
- **Branch-options are parallel.** When a round needs to compare directions, it spawns `options/two-pane/`, `options/stacked/`, etc. They are explored side by side, then **one is chosen** and folds forward into the next round's mainline. The others stay in the folder as a record (and in the guide's decisions).

## Folder shape (extends today's `round-N/`)

```
design-system/lofi/
├── KICKOFF.md              # lo-fi brief / kickoff notes — seeds round 1
├── round-1/
│   ├── CLAUDE.md           # round agreement: links design system + /wireframe-index + kickoff
│   ├── brief.md            # what this round resolves (from PM)
│   ├── context/            # dropped transcripts/scripts — mined for the "why"
│   ├── wireframes/         # the actual HTML wireframe files (built from WireframeDesignSystem)
│   ├── index.html          # simple gallery linking the wireframes (Launch target)
│   └── WIREFRAME-GUIDE.html# generated by /wireframe-index — the transparency/decisions doc
├── round-2/
│   ├── CLAUDE.md           # references round-1 as the base to iterate on
│   ├── …
│   ├── WIREFRAME-GUIDE.html# its "Changes from Round 1" section is the cross-round changelog
│   └── options/            # parallel branch-options for this round
│       ├── two-pane/       #   Direction A: { CLAUDE.md, index.html, wireframes/ }
│       └── stacked/        #   Direction B
└── HANDOFF.md              # generated at the lo-fi → hi-fi boundary
```

Today's `createLofiRound` (`server/projects.ts` ~:689) already makes `round-N/` with `index.html`, `notes.md`, and `wireframes/.gitkeep`. The redesign extends that with `CLAUDE.md`, `brief.md`, `context/`, `options/`, and the project-level `KICKOFF.md`. The per-round `WIREFRAME-GUIDE.html` is **produced by the `/wireframe-index` skill**, not scaffolded empty — the guide skill already handles cross-round changelog via its "Changes from Previous Round" section, so there's no separate `CHANGELOG.md`.

## Round lifecycle (drives the UI states)

1. **Drafted** — folder + `CLAUDE.md` exist, `wireframes/` empty.
   - Hub shows **Send to Claude** (copies the round path + a ready prompt). **Launch disabled.**
2. **Built** — `wireframes/` has real files (detected by file presence, like the existing `pathExists` checks).
   - **Launch enabled** → opens `file://…/index.html` in the browser. **No server, no port.**
3. **Documented** — `/wireframe-index` has generated `WIREFRAME-GUIDE.html` (round context, decisions, changes, open questions).
   - The Hub's **Guide / Decisions** affordance opens it. Round can spawn **branch-options** or the **next round**.
4. **Chosen** — one option selected; folds into the next round's mainline.

Status is a derived signal, not a stored flag — the Hub infers it from folder contents (mirrors how hi-fi infers "running" / "behind" from git + process state). Presence of `WIREFRAME-GUIDE.html` ⇒ Documented.

## Lo-fi vs hi-fi — why it "looks a little different"

| | Hi-fi (exists today) | Lo-fi (this redesign) |
|---|---|---|
| Unit | Variant = git worktree on a branch | Round = plain folder |
| Iterate | New variant forked at a version hash | New round folder referencing the previous round |
| Branch | n/a (variants *are* the fork) | Branch-options = sibling `options/*` folders |
| Launch | Dev server on a port (`yarn start`) | Open static HTML (`file://`) — **no server** |
| Document | Git log / commits | `/wireframe-index` → `WIREFRAME-GUIDE.html` |
| Promote | `git merge` + version stamp | "Choose option" folds folder forward; "Promote to hi-fi" wires the handoff |

The **timeline visual** mirrors hi-fi (a solid mainline of rounds + dashed branch-options) for UI consistency, but the mechanics stay lightweight and document-centric.

## Design-system + skill linkage

Every project should know both pieces without manual pointing:

- **`renderClaudeMd`** (`server/claudemd.ts:8`) gains a section pointing at `~/Documents/WireframeDesignSystem/` (build *from* this) and at the `/wireframe-index` skill (document *with* this — run it after each round).
- Each **round `CLAUDE.md`** repeats both pointers in-context and adds: the previous round (base to iterate on), the kickoff, this round's `brief.md`, the `context/` drop folder, and `resources/`.
- **No new skill to author** — `wireframe-index` already exists and is enabled. The work is wiring the Hub to invoke it and surface its output.

## Lo-fi → hi-fi handoff (both mechanisms)

On **Promote to hi-fi**:

1. **Auto-reference in CLAUDE.md.** The hi-fi project's `CLAUDE.md` auto-injects references to: the **chosen round's wireframes**, its `WIREFRAME-GUIDE.html` (decisions + rationale), `KICKOFF.md`, and the design system. Claude sees the lo-fi truth with zero manual pointing.
2. **Generated handoff doc.** Produce `lofi/HANDOFF.md` (via the `/handoff-doc` skill) at the boundary, referenced by the hi-fi CLAUDE.md. (`/handoff-doc` is the build-spec engine; `/wireframe-index` is the per-round design-rationale engine — they compose.)

The intent is a durable connection: every lo-fi round is built knowing it may become hi-fi, and the hi-fi build always has the lo-fi rationale on hand.

## The lo-fi section UI (what the wireframe shows)

In the **Prototypes tab**, replacing `TimelinePlaceholder`:

- **Round timeline** — a solid mainline R1→R2→R3 with dashed branch-options ("two-pane", "stacked") hanging off R2; the active round is highlighted.
- **Round card** — shows status (Drafted / Built / Documented) and actions: **Send to Claude**, **↗ Launch** (enabled only when Built), **⌥ Branch option**, **📓 Guide** (opens `WIREFRAME-GUIDE.html`).
- **Guide / decisions panel** — surfaces the round's `WIREFRAME-GUIDE.html` content: decisions, changes-from-previous, open questions. The transparency layer.
- **Kickoff / brief** entry point — seeds round 1.
- **Promote to hi-fi** affordance — shows exactly what gets auto-wired into the hi-fi CLAUDE.md.
- A short **legend** calling out lo-fi-specific behavior (static-HTML launch, options = folders, guide = `/wireframe-index`).
