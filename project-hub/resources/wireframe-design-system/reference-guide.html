Gravitate Wireframe Design System

A comprehensive, pure HTML/CSS design system for creating consistent, interactive low-fidelity wireframes for Gravitate applications.

What is this?

This design system enables rapid creation of wireframes that communicate layout, hierarchy, and interaction patterns without needing a build system or JavaScript framework.

Key Features

  • 60+ component variations across 8 categories
  • 6 full-page pattern templates for common layouts
  • Pure HTML/CSS - no build step required
  • Interactive states - hover, focus, disabled, all via CSS
  • Maps to Excalibrr - clean transition to production

Design Principles

  • Structure over style - Focus on layout and hierarchy
  • Semantic naming - Classes describe purpose, not appearance
  • Consistent patterns - Same approach across all components
  • Low-fidelity friendly - Works without images or icons

Quick Start

Get up and running in under a minute.

1. Include the CSS

<link rel="stylesheet" href="wireframe.css">

2. Create a basic page

<!DOCTYPE html>
<html>
<head>
  <link rel="stylesheet" href="wireframe.css">
</head>
<body>
  <div class="wf-page">
    <header class="wf-page-header">
      <h1 class="wf-text wf-text-h1">My Wireframe</h1>
      <button class="wf-button wf-button-primary">Action</button>
    </header>
    <main class="wf-page-content">
      <div class="wf-card">
        <div class="wf-card-body">
          Your content here
        </div>
      </div>
    </main>
  </div>
</body>
</html>

3. Browse this guide

Use the sidebar to explore all available components, see live previews, and copy code snippets.

How to Use

Understanding the class naming conventions and patterns.

Class Naming Convention

All classes use the wf- prefix (wireframe):

Pattern Example Description
wf-{component} wf-button Base component class
wf-{component}-{variant} wf-button-primary Component variant
wf-{component}-{size} wf-button-sm Size modifier
wf-{utility} wf-gap-4 Utility class

Combining Classes

<button class="wf-button wf-button-primary wf-button-lg">
  Large Primary Button
</button>
