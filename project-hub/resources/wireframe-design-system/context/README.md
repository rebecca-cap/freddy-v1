# Context Folder

This folder is part of the Ralph Wiggum context management system for the WireframeDesignSystem project.

## Purpose
The context folder stores session-specific information, notes, and artifacts that help maintain project continuity across work sessions.

## What Goes Here
- Session notes and summaries
- Research findings
- Decision logs
- Temporary work-in-progress files
- Reference materials

## How It Works
At the end of each session, relevant information from this folder is consolidated into the main PROJECT_CONTEXT.md file. The PROJECT_CONTEXT.md file serves as the source of truth for overall project state.

## Workflow
1. During a session: Store temporary notes and artifacts here
2. Session end: Review and update PROJECT_CONTEXT.md with key decisions and progress
3. Session start: Read PROJECT_CONTEXT.md for full context, then reference this folder for recent session details
